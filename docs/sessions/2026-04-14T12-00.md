## 1. Goals

- Evaluate and perform a migration from "ticker" to "symbol" to align with industrial standards.
- Rename core entities (`Ticker` -> `Symbol`, `Symbol` -> `Security`).
- Provide a migration guide for users of the library.

## 2. Work Done

- Searched codebase to identify current usage of "ticker" and "symbol".
- Renamed `Ticker` class to `Symbol`.
- Renamed `Symbol` class to `Security`.
- Updated fields in `Security`, `History`, `PriceVerificationError`, and `SecurityCriteria`.
- Updated `DataSource` protocol interfaces to use `symbol` parameters and `Security` types.
- Updated CLI arguments in `SearchArgs` and `HistoryArgs` to use `--symbol` instead of `--ticker`.
- Updated tests (`test_models.py`, `test_cli_models.py`, `test_price_error.py`, `smoke_test.py`) and ran `uv run pytest` to confirm they pass.
- Updated examples (`basic_usage.py`, `custom_datasource.py`).
- Updated documentation (`README.md`, `GEMINI.md`, `CHANGELOG.md` bumping version to `0.2.0`).
- Created a `MIGRATION_GUIDE.md` for library users.
- Updated `pyproject.toml` version to `0.2.0`.

## 3. Decisions Made

| Decision | Rationale |
|----------|-----------|
| Perform full rename (`Ticker` to `Symbol`, `Symbol` to `Security`) | Better alignment with industrial standards (FIX, Yahoo Finance) where "Symbol" is the string identifier and "Security" or "Instrument" is the asset object. Resolves internal inconsistencies. |
| Bump version to 0.2.0 | This is a breaking change affecting core model names and interfaces. |

## 4. Decisions Refused / Alternatives Rejected

| Rejected Approach | Reason |
|-------------------|--------|
| Just standardizing field names or aliasing without renaming classes | Avoids breaking changes, but leaves the core models with confusing and non-standard names (e.g., `Symbol` containing a `symbol` string). Full migration provides better long-term clarity. |
